# Security Policy

## 📢 Reporting a Vulnerability

We take security, code quality, and reliability seriously in the **Java-Core-Concepts** repository.

If you discover any security vulnerability, bug, or issue, please follow these steps:

1. **Do not** publicly disclose the issue until it has been reviewed.
2. Report the issue via:
   - [GitHub Issues](https://github.com/PrateekRaj8125/Java-Core-Concepts/issues) (for general bugs)
   - [GitHub Security Advisory](https://github.com/PrateekRaj8125/Java-Core-Concepts/security/advisories) (for sensitive vulnerabilities)
   - Email: <prateek.raj.indt@gmail.com>
3. Include the following details:
   - Steps to reproduce the issue
   - Description of the vulnerability
   - Possible impact
   - Suggested fix (if available)

---

## 🔔 What to Expect

- Acknowledgment within **48 hours**
- Review and status update within **7 days**
- Fix implemented if the issue is valid
- Credit provided to contributors (optional)

---

## ✅ Scope

This repository primarily contains **educational Java programs**, so security risks are minimal. However, reporting issues related to:

- Incorrect logic
- Unsafe coding practices
- File handling vulnerabilities

is encouraged.
